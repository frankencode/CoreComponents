Detailed Description

The Utf8Walker allows iterating Unicode characters over any zero terminated byte sequence. The walker will always halt at the string boundaries. If stepping over the string boundary the walker will switch to valid() == false and start delivering zero characters.

If placed at a string's terminating zero character it is possible to step backwards into the string. In other words '(Utf8Walter(data, data + len) - 1).getChar()' is a valid construct and will deliver the last Unicode character of the string.

Illegal code sequences are overrun without error. In forward iteration corrupted code prefixes may lead to at maximum one correct character overrun and up to 3 additional corrupt characters delivered. Any bit error outside the code prefixes will lead to at most one broken character delivered, with one exception: Switching to all bits zero. A zero byte is always string terminating.

A walker object equals in size the size of 2 pointers and can be passed by value.
